# Feedback Loops — Post Haste

## DATA -> LEARNING -> ACTION

### Pre-Launch Research Findings

**DATA:** Sorting is inherently satisfying as a mobile mechanic (Water Sort Puzzle genre proves this)
**LEARNING:** The satisfaction comes from pattern recognition + visual order, not dexterity
**ACTION:** Sort mechanic should reward speed of DECISION, not speed of thumb movement

**DATA:** Arcade idle games use joystick + auto-proximity exclusively — no buttons
**LEARNING:** Adding ANY button to the joystick mode breaks the one-thumb contract
**ACTION:** Sorting desk must be a SEPARATE mode (character stops moving, swipe UI appears)

**DATA:** Stack-on-head visual is the #1 marketing creative element in arcade idle
**LEARNING:** The wobbling stack IS the game feel — it communicates progress and is inherently funny
**ACTION:** Mail/parcels must stack visually on character. Bigger parcels = more wobble.

**DATA:** My Perfect Hotel uses "stand still to complete" vs My Mini Mart's "instant transfer"
**LEARNING:** Dwell-to-complete creates more visible opportunity cost per decision
**ACTION:** Sorting desk uses dwell (you commit to sorting a batch), pickups use instant transfer
