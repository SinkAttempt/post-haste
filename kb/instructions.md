# Instructions — Post Haste

The living rulebook. Rules derived from research, playtesting, and feedback.

## Control Rules
- NEVER add buttons to the joystick movement mode. One thumb, one joystick, zero buttons.
- Sorting desk is a SEPARATE interaction mode — character stops, swipe UI appears.
- All station interactions outside sorting use auto-proximity (walk near = auto-perform).
- Interaction zone must be in bottom 60% of screen (thumb reach).

## Visual Rules
- Mail/parcels MUST stack visually on character's head. This is the core game feel.
- Bigger parcels = more wobble. Stack height = comedy + capacity feedback.
- Sort destinations should be colour-coded. Brain processes colour pre-consciously.
- "Chaos to order" must be VISIBLE — messy incoming pile vs neat sorted output.

## Progression Rules
- Sorting complexity scales through item variety, NOT control complexity.
- Early: sort by colour (2 bins). Mid: sort by destination (4 bins). Late: multi-criteria.
- Carry capacity upgrade is the PRIMARY upgrade hook (visible, satisfying, marketable).
- Day-based structure for natural session boundaries.

## Design Principles
- The game is a ROUTING PUZZLE disguised as a job sim. Where you go next IS the skill.
- Active play must always be faster than idle/automation (reason to stay engaged).
- Priority crunch = fun. Player should always have 3 things to do and only 2 hands.
- Escalation through VOLUME (more mail per minute), not complexity (more button types).
